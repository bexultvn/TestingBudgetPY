# План тестирования – BudgetPy

## 1. Область тестирования
Тестирование охватывает функциональную логику консольного приложения BudgetPy.
Основное внимание уделяется корректности бизнес-логики и работе с данными.

---

## 2. Типы тестирования
- Модульное тестирование (Unit testing)
- Интеграционное тестирование(Integration testing)

---

## 3. Объекты тестирования
- Управление балансом (Account)
- Транзакции (Transactions)
- Копилки (Savings)
- Категории транзакций (Categories)
- Хранение данных в JSON-файле (Storage)

---

## 4. Функции, подлежащие тестированию

### 4.1 Управление балансом
- Создание счёта с начальным балансом
- Пополнение баланса
- Снятие средств
- Обработка недостаточного баланса

### 4.2 Транзакции
- Сохранение доходных и расходных транзакций
- Корректность суммы и категории транзакции
- Фильтрация транзакций по категории

### 4.3 Копилки
- Создание цели накопления
- Добавление средств в цель
- Отслеживание прогресса накопления
- Обработка некорректных целевых сумм

### 4.4 Категории
- Создание пользовательских категорий
- Предотвращение создания дублирующихся категорий
- Получение списка доступных категорий

### 4.5 Хранение данных
- Сохранение данных в JSON-файл
- Загрузка данных из JSON-файла
- Корректная работа при отсутствии файла данных

---

## 5. Не подлежит тестированию
- Пользовательский интерфейс консоли
- Производительность приложения
- Многопользовательские сценарии

---

## 6. Инструменты тестирования
- Python 3.x
- Pytest
- Pytest fixtures
- Mock / Fake объекты для изоляции зависимостей

---

## 7. Тестовая среда
- Локальная среда разработки
- Виртуальное окружение Python (venv)
- macOS

---

## 8. Критерии начала тестирования
- Реализован код приложения
- Определены требования к продукту
- Настроена тестовая среда
- Установлены необходимые зависимости

---

## 9. Критерии завершения тестирования
- Все запланированные тесты выполнены
- Все unit-тесты проходят успешно
- Критические дефекты отсутствуют
- Основная функциональность покрыта автотестами

---

## 10. Риски
- Отсутствие валидации входных данных
- Жёсткая зависимость бизнес-логики от хранилища данных
- Ошибки при работе с файловой системой

---

## 11. Артефакты тестирования
- Автоматизированные тесты (pytest)
- Файл требований (`requirements.md`)
- План тестирования (`test_plan.md`)
- Отчёты о выполнении тестов (pytest output)
