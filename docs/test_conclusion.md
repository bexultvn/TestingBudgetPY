## Заключение по тестированию

Тестирование приложения BudgetPy было выполнено на основе заранее определённых функциональных и интеграционных требований.

В рамках проекта были проведены следующие виды тестирования:
- модульное тестирование бизнес-логики;
- интеграционное тестирование взаимодействия модулей и сохранения данных;
- негативное тестирование некорректных входных данных;
- тестирование граничных значений.

Все ключевые функции приложения покрыты автоматизированными тестами с использованием pytest.  
В ходе интеграционного тестирования были выявлены и исправлены дефекты, связанные с инициализацией данных и сохранением состояния между запусками приложения.

На момент завершения тестирования:
- все запланированные тесты успешно выполняются;
- критические дефекты отсутствуют;
- поведение приложения соответствует заявленным требованиям.

Приложение считается стабильным и готовым к дальнейшему развитию либо использованию в качестве демонстрационного проекта по QA Automation.
