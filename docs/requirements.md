# Требования к продукту – BudgetPy

## 1. Общее описание
BudgetPy — это консольное приложение для управления личным бюджетом одного пользователя.  
Данные хранятся в JSON-файле.

---

## 2. Управление балансом

### FR-1
Система должна позволять создавать счёт с начальным балансом.

### FR-2
Система должна позволять пополнять баланс.

### FR-3
Система должна позволять снимать средства при достаточном балансе.

### FR-4
Система не должна позволять снимать сумму, превышающую текущий баланс.

---

## 3. Транзакции

### FR-5
Система должна сохранять транзакции с указанием суммы и категории.

### FR-6
Система должна позволять фильтровать транзакции по категории.

---

## 4. Копилки (накопления)

### FR-7
Система должна позволять создавать цель накопления с целевой суммой.

### FR-8
Система должна позволять добавлять средства в существующую цель накопления.

### FR-9
Система должна отслеживать текущий прогресс накопления для каждой цели.

### FR-10
Система не должна позволять создавать цель накопления с целевой суммой меньше или равной нулю.

---

## 5. Категории

### FR-11
Система должна позволять создавать пользовательские категории транзакций.

### FR-12
Система не должна позволять создавать дублирующиеся категории.

### FR-13
Система должна позволять получать список всех доступных категорий.

---

## 6. Хранение данных

### FR-14
Система должна сохранять данные в JSON-файл.

### FR-15
Система должна загружать данные из JSON-файла при запуске приложения.

---

## 7. Интеграционные требования

### FR-16
Система должна сохранять состояние счёта (баланс и транзакции) и корректно восстанавливать его после перезапуска приложения.

### FR-17
Система должна корректно работать при совместном использовании модулей управления балансом и хранения данных.

### FR-18
Система должна обеспечивать корректное взаимодействие модулей транзакций и категорий при загрузке данных из JSON-файла.

### FR-19
Система должна сохранять и восстанавливать данные копилок (накоплений) при перезапуске приложения.

### FR-20
Система должна обеспечивать целостность данных при последовательных операциях записи и чтения из JSON-файла.
